01 // PRODUCT OVERVIEW

What is the Translation Preprocessor?

The OAOA.dev Translation Preprocessor is an ASP.NET Core microservice designed to process incoming mixed-language text streams (such as chat messages, filenames, and documents) from OSINT signal collection platforms like TeleGather.

Instead of naively sending large text payloads to external translation APIs or local LLMs—which causes high latency, token waste, and potential distortion of specialized English terms—this microservice intelligently segments incoming text, verifies existing English content against an embedded 73k-word dictionary, and dispatches only the non-English segments for parallel translation.

Why We Built This

Incoming intelligence streams often contain mixed languages (e.g. English technical terminology mixed with Russian, Spanish, French, or Chinese text). Translating entire messages through LLMs or NMT models introduces unnecessary overhead.

Benefit 01

Reduces Payload Size

Bypasses 60–90% of already-English text segments automatically.

Benefit 02

Preserves Original Context

Keeps English technical terms, code snippets, and proper nouns untouched.

Benefit 03

Accelerates Throughput

Translates non-English segments concurrently in parallel batches.

Benefit 04

Multi-Engine Routing

Supports seamless toggling between LibreTranslate (fast NMT) and TranslateGemma via Ollama.

Stage 01

1. Tokenization & Sentence Parsing

Incoming text is tokenized into sentences and words using the C# Catalyst NLP engine, preserving whitespace and structural formatting.

Stage 02

2. English Dictionary Bypass

Each token is cross-referenced against an embedded 73,000-word English lexicon. Sentences and clauses evaluated as predominantly English bypass translation entirely.

Stage 03

3. Script Grouping & Batching

Non-English segments are grouped by script (Latin, Cyrillic, CJK, Arabic, etc.) and dispatched concurrently via asynchronous HTTP requests (Task.WhenAll).

Stage 04

4. Text Assembly

Translated segments are spliced back into their exact original locations, preserving all line breaks, spaces, and punctuation.

04 // API REFERENCE

Minimal API Specification

POST /translate-mixed

Process and translate mixed-language input text.

Request Body Example

{
  "Text": "Urgent update: Этот сервер работает отлично, но нам нужно настроить бэкап баз данных. Please schedule this backup daily at 2:00 AM UTC.",
  "TargetLang": "en",
  "Provider": "libretranslate"
}

Request Parameters

Parameter Type Required Default Description
Text string Yes — Mixed-language input text to process and translate.
TargetLang string No "en" Target language code (currently supports "en").
Provider string No "libretranslate" Translation provider: "libretranslate" or "translategemma".

Response Body (200 OK)

{
  "translatedText": "Urgent update: This server works great, but we need to set up a database backup. Please schedule this backup daily at 2:00 AM UTC."
}
