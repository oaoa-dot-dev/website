Test delegated mail send via Microsoft 365 using the OAuth 2.0 Device Code flow. No redirect URI required.

How This Works This tool uses the OAuth 2.0 Device Authorization Grant (RFC 8628). Enter your tenant endpoint, the API server requests a device code from Microsoft, and you authenticate at login.microsoftonline.com/devicelogin on any device using the displayed code. Once authorised, mail is sent via the Microsoft Graph API (/v1.0/me/sendMail) using delegated Mail.Send permissions. Your credentials are never stored — only a short-lived session token is held server-side.
